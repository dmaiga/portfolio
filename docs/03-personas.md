# 03 — Personas

## Le pair (cible prioritaire)
- **Qui.** Un collègue, un camarade ou un passionné du même métier que moi.
- **Cherche en 30 s.** Un projet ou du code concret à explorer : pour s'inspirer, cloner, améliorer, faire une revue de code.
- **Reste** longtemps s'il accroche : il descend dans le code, l'architecture, les artefacts de cadrage.
- **Part si** rien n'est creusable — ni dépôt, ni démo, ni détail technique accessible.

## Le recruteur (levier)
- **Qui.** Le plus souvent **non-tech** : dans l'économie actuelle, c'est lui qui pilote toute la chaîne de recrutement. On conçoit d'abord pour lui — sans rendre le site illisible pour autant.
- **Cherche en 30 s.** Se faire une idée du profil et des compétences, vite, sans jargon : une documentation de cadrage écrite et lisible (selon le projet : cahier des charges, périmètre, TDR… — liste indicative, adaptée au cas).
- **Double besoin.** Le site doit servir **les deux** : le non-tech via la doc écrite ; un évaluateur tech via les liens GitHub, démo en ligne, `docs/`, `rules/` — pour exprimer ce qu'un CV ne montre pas.
- **Part si** il se heurte d'emblée à un mur technique illisible, ou à l'inverse ne trouve rien de concret à vérifier.
